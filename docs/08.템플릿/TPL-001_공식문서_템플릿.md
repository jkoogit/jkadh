## 1. 목적

본 문서는 JKADH 공식 문서를 새로 작성할 때 사용할 기본 구조를 정의한다.

공식 문서 템플릿은 문서마다 반복되는 Header, 본문 구조, 관련 문서, 작업 이력 형식을 통일하기 위한 기준이다.

## 2. 적용 범위

본 템플릿은 다음 공식 문서 유형에 적용한다.

- 시작 문서
- 프로젝트 문서
- 조직 문서
- 정책 문서
- 결정 문서
- 설계 문서
- 기획 문서
- 로드맵 문서
- 운영 문서
- 점검 문서
- 회고 문서
- 참고 문서

Backlog 문서와 폴더 `README.md` 인덱스 문서는 별도 구조를 사용한다.

## 3. 파일명

파일명은 다음 형식을 사용한다.

```text
{문서ID}_{문서명}.md
```

예:

```text
POL-001_문서관리방안.md
DSN-001_Harness_운영_모델.md
RDM-001_단계적_추진방향.md
```

## 4. Header

공식 문서는 제목 바로 아래에 Markdown 표 형식 Header를 둔다.

```markdown

| 항목 | 값 |
|---|---|
| 문서 ID | {문서ID} |
```

- 관련 Work Order
- 관련 ADR
- 원천 세션
- 승인자

Issue와 PR은 Header에 두지 않고 작업 이력에 기록한다.
