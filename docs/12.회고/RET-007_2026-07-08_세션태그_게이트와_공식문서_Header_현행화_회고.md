## 1. 목적

본 문서는 2026-07-08 세션에서 진행한 HCP 실행 게이트 보강, 태스크승급 시 Issue 미종료 기준 보강, 공식문서 Header 표 형식 전체 현행화 작업을 회고한다.

이번 회고의 목적은 절차가 흔들린 지점을 실제 문서 규칙으로 고정하고, 다음 세션에서 이어갈 낮은 우선순위 Backlog만 남기는 것이다.

## 4. 잘된 점

- `#태스크승급`이 Issue 종료까지 수행해버린 문제를 바로 문서 규칙으로 보정했다.
- 보정 후 다음 승급부터 Issue를 열어둔 채 PR 머지와 브랜치 승급만 수행하는 흐름이 실제로 적용되었다.
- BLG-021 작업에서 README, Backlog, Backlog 전용 템플릿을 제외하고 공식 문서 본문형 파일만 대상으로 제한했다.
- 공식 문서 Header 변환은 본문 의미 변경 없이 Header 형식과 작업 이력만 현행화했다.

## 5. 문제와 보정

초기 `#태스크승급`에서 Issue #37을 함께 종료했다.

이는 승급과 Issue 종료가 같은 절차처럼 처리된 문제였다. 이후 Issue #39를 통해 `#태스크승급`은 Issue를 종료하지 않고, Issue 종료는 `#태스크정리` 또는 `#세션정리`에서 완료 조건과 후속 Backlog를 별도로 확인한 뒤 수행하도록 문서화했다.

공식문서 Header 현행화는 변경 파일 수가 많아 diff가 커졌다. 이를 보정하기 위해 변환 대상을 본문형 공식 문서로 제한하고, `git diff --check`, 잔여 blockquote Header 검색, 제외 대상 변경 여부를 검증했다.
