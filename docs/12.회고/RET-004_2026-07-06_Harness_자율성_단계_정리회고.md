## 1. 목적

본 문서는 2026-07-06 세션에서 진행한 Harness 자율성 단계 정리 작업을 회고하고, 다음 세션에서 이어갈 작업 후보와 운영 개선점을 남긴다.

## 2. 세션 요약

이번 세션은 이전 회고에서 권장한 `BLG-011 Harness 자율성 단계 정리`를 실제 Issue로 승격해 진행했다.

핵심 결과는 Harness가 어디까지 판단할 수 있고, 어디서 사람 승인이나 리뷰로 전환해야 하는지 로드맵 수준에서 정리한 것이다. 또한 작업 도중 확인된 세션 기준 Issue 운영, 문서 탈고, 끼어든 주제 분류 기준도 정책 문서에 보강했다.

## 4. 잘된 점

- 이전 회고의 다음 세션 권장 시작점을 실제 작업으로 연결했다.
- Harness 자율성을 한 번에 자동화하지 않고 `A0`부터 `A5`까지 단계화했다.
- 승인 상태와 결과 수용 상태를 분리해 `Approval Pass`와 `Accept Candidate`가 섞이지 않도록 했다.
- Token 제약을 실패가 아니라 축소, 분할, 재배치, 중단 판단 신호로 정리했다.
- 문서 작업 완료 전 탈고 게이트를 정책에 넣어 문맥, 충돌, 중복, 쉬운 표현을 확인하도록 했다.
- 현재 태스크와 맞지 않는 주제가 들어왔을 때 `[번외]`, Backlog, 새 Issue 후보로 분류하는 기준을 남겼다.
- PR #15 병합 후 `dev`, `stg`, `main`이 같은 커밋과 tree를 가리키는지 확인했다.

## 5. 아쉬운 점

- 세션 마무리 요청이 온 뒤에야 회고 Issue를 별도로 만들었다. 다음부터는 승급 직후 세션 회고 필요 여부를 먼저 제안하는 편이 자연스럽다.
- 회고는 별도 Issue보다 작업 중인 Issue의 마무리 태스크로 처리하는 편이 더 자연스럽다. 이 개선점은 `BLG-015`로 남겼다.
- 세션 기준 Issue 운영 방식은 일부 기준만 정책에 반영했고, 전체 운영 모델 정리는 `BLG-014`로 남았다.
- 회고 문서 작성도 하나의 변경 작업이므로 별도 Issue와 PR이 필요해 세션 종료 절차가 다소 길어졌다.

## 6. 개선 사항

- `#태스크정리` 단계에서 문서 작업이 있었다면 탈고 결과를 PR 본문에 반드시 남긴다.
- 현재 태스크와 맞지 않는 주제가 들어오면 바로 처리하지 않고 현재 Issue 반영, `[번외]`, Backlog, 새 Issue 후보 중 하나로 먼저 분류한다.
- `#세션마무리` 요청 시에는 새 회고 문서를 만들지, 대화 요약으로만 마무리할지 먼저 현재 작업 상태를 기준으로 판단한다.
- 다음 세션부터 회고는 별도 Issue를 만들지 않고 작업 중인 Issue의 마무리 태스크로 정리한다.
- 승급 완료 후에는 다음 작업을 바로 시작하기보다 남은 Backlog와 브랜치 상태를 먼저 보고한다.
