## 1. 내용

`#세션정리`가 보고만으로 끝나지 않도록 실행 체크리스트와 완료 조건을 보강한다.

또한 `#태스크시작`, `#태스크정리`, `#태스크승급`, `#세션정리`를 논스톱으로 진행할 수 있는 조건과 중단 기준을 정리한다.

## 2. 발생 배경

Issue #19 세션을 정리하면서 저장소 상태와 남은 Backlog는 보고했지만, 회고 문서 작성과 Issue 현행화 여부 같은 세션정리 산출물 점검이 누락되었다.

세션정리도 다른 태그처럼 실행 체크리스트와 완료 조건이 있어야 흔들리지 않는다.

## 3. 처리 기준

- `#세션정리`는 저장소 상태, 현재 Issue, 세션명, 회고, Backlog, 다음 세션, 미처리 항목을 확인한다.
- 회고는 저장소 문서 또는 Issue 댓글처럼 다음 세션에서 다시 찾을 수 있는 위치에 남긴다.
- 논스톱 진행은 사용자가 네 단계의 연속 실행과 중간 확인 생략을 명시한 경우에만 허용한다.
- 논스톱 진행 중 충돌, 실패, 새 판단 지점, 누락 산출물이 확인되면 중단하고 확인한다.

## 4. 기대 효과

- 세션정리가 대화 보고만으로 끝나는 문제를 줄인다.
- PR, 승급, 회고, Issue 현행화가 누락되지 않는다.
- 명시 승인된 고속 진행과 기본 확인 절차를 구분할 수 있다.
