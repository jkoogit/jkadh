## 1. 내용

사용자 주문과 저장소 표준 절차가 충돌하거나 해석이 갈릴 때 에이전트가 실행 전에 확인하는 기준을 정리한다.

속도, 마무리, 다음 세션 전환 같은 자연어 주문은 상황에 따라 의사결정을 빠르게 정리하라는 뜻일 수 있다. 이를 PR 병합, 승급, Issue 종료 같은 절차 생략 승인으로 해석하면 관리 투명성이 떨어진다.

## 2. 발생 배경

BLG-019와 BLG-020을 처리하는 과정에서 사용자의 “후딱 처리하고 다음세션으로 가자”는 주문을 에이전트가 PR 병합과 승급까지 진행하라는 뜻으로 해석했다.

사용자의 의도는 결정을 빠르게 정리해 마무리하자는 것이었고, 표준 절차를 건너뛰라는 승인이 아니었다. 주문과 프로세스가 충돌할 때는 확인 후 진행하는 기준이 필요하다.

## 3. 처리 기준

- 사용자 주문과 표준 절차가 충돌하거나 해석이 갈리면 실행 전에 확인한다.
- 명시 태그가 필요한 단계는 자연어 주문만으로 자동 실행하지 않는다.
- PR Ready 전환, `dev` 머지, `stg` 또는 `main` 승급처럼 저장소 상태를 바꾸는 단계는 명확한 승인 후 진행한다.
- 이미 잘못 진행된 경우에는 되돌리기보다 별도 태스크로 원인과 보정 기준을 문서화한다.

## 4. 기대 효과

- 사용자 의도와 에이전트 실행 범위가 어긋나는 일을 줄인다.
- 빠른 진행 요청이 절차 생략 승인으로 오해되지 않는다.
- PR, 승급, Issue 종료 같은 상태 변경 작업의 통제 기준이 명확해진다.
