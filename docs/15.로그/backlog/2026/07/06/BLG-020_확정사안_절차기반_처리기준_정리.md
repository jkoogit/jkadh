## 1. 내용

결론이 이미 확정된 사안이라도 주제와 영향 범위에 맞는 처리 절차를 거쳐 반영하는 기준을 정리한다.

이 기준은 모든 에이전트의 공통 역할로 다루어야 하며, 업무 처리가 체계적으로 해결되도록 `ORG-001`, `STA-002`, `POL-003`, Harness 운영 모델에 반영할지 검토한다.

## 2. 발생 배경

PR 분리와 같은 PR 포함 판단 기준을 논의하면서 결론이 명확해 보이는 사안도 바로 문서화하거나 현재 PR에 섞어 넣으면 관리 투명성이 떨어질 수 있다는 점이 확인되었다.

특히 Harness 프로세스도 향후 같은 문제를 만날 수 있다. Harness가 결론만 보고 바로 실행하면, 왜 이 작업이 현재 PR에 포함되었는지, 별도 PR로 분리되었는지, Backlog로 남겼는지 추적하기 어렵다.

따라서 결론의 명확성과 별개로, 주제에 맞는 처리 절차를 거치는 기준이 필요하다.

## 3. 기대 효과

- 에이전트가 결론을 성급히 실행하지 않고 절차와 영향 범위를 먼저 확인한다.
- 현재 PR 포함, 별도 PR 분리, Backlog 등록, 결정 문서 작성의 기준이 투명해진다.
- Harness가 Issue, PR, Backlog, Report 흐름을 판단할 때 같은 기준을 사용할 수 있다.
- 모든 에이전트가 업무 처리의 추적 가능성과 설명 가능성을 유지한다.

## 4. 처리 기준

다음 기준을 문서화한다.

- 결론이 확정된 사안도 영향 범위와 문서 성격을 확인한다.
- 현재 PR의 정확성을 위해 필요한 보정인지, 독립 판단이 필요한 새 기준인지 구분한다.
- 모든 에이전트는 중요한 기준 변경을 즉시 실행하기보다 현재 Issue 반영, `[번외]`, Backlog, 새 Issue 후보, 결정 문서 중 하나로 분류한다.
- `#태스크시작` 또는 `#태스크정리`에서 해당 사안의 처리 경로를 명시한다.
- 시작 지침에는 “결론 확정 여부와 별개로 처리 절차를 확인한다”는 원칙을 포함할지 검토한다.
- Harness 운영 모델에는 같은 판단을 Work Order Review, Classify, Decompose, Backlog, PR 판단 지점과 연결할지 검토한다.
