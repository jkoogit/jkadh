## 1. 내용

`#태스크정리`에서 PR을 만들 때 GitHub 자동 종료 키워드로 Issue를 닫지 않는 기준을 정리한다.

세션 또는 업무 단위 Issue 안에서 여러 PR을 연결할 수 있다면, 개별 정리 PR이 머지되는 시점이 곧 Issue 종료 시점이 아닐 수 있다. 따라서 PR 본문에는 `Closes`, `Fixes`, `Resolves` 같은 자동 종료 키워드 대신 `Related #이슈번호`처럼 추적 가능한 연결만 남기고, Issue 종료는 별도 조건으로 판단하는 기준이 필요하다.

## 2. 발생 배경

BLG-014와 BLG-015를 정리하는 PR #20을 만들 때 PR 본문에 `Closes #19`를 포함했다. 그러나 사용자가 태스크 정리 시 Issue를 바로 닫으면 안 된다고 지적했고, PR 본문은 `Related #19`로 보정했다.

## 3. 기대 효과

- 세션 또는 업무 단위 Issue가 중간 PR 머지로 조기 종료되는 것을 막는다.
- 같은 Issue 안에서 여러 PR을 연결하는 운영방식과 충돌하지 않는다.
- Issue 종료 조건을 PR 머지와 분리해 더 명확히 판단할 수 있다.

## 4. 처리 기준

`POL-002`, `POL-003`에서 다음 기준을 정리한다.

- `#태스크정리` PR 본문에는 기본적으로 자동 종료 키워드를 쓰지 않는다.
- PR과 Issue 연결은 `Related #이슈번호` 또는 명시 링크로 남긴다.
- Issue 종료는 세션 종료, 승급 완료, 완료 조건 충족 여부를 확인한 뒤 별도로 판단한다.
- 단일 PR로 닫는 것이 명확한 작은 Issue에만 자동 종료 키워드 사용을 허용할지 검토한다.
