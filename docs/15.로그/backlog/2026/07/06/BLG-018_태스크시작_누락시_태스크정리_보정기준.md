## 1. 내용

업무 태스크는 `#태스크시작`으로 시작하는 것을 원칙으로 하되, 대화 중 백로그, 번외사항, 작은 보정이 먼저 진행된 경우 `#태스크정리`에서 사후 보정하는 기준을 정리한다.

## 2. 발생 배경

작업을 진행하다 보면 명시적인 `#태스크시작` 없이 백로그나 번외사항이 대화 중 자연스럽게 튀어나와 처리되는 경우가 있다. 사용자는 업무적인 태스크는 `#태스크시작`으로 시작하고 싶지만, 현실적으로 중간 항목을 모두 즉시 멈추고 절차화하기는 어렵다고 보았다.

따라서 `#태스크시작` 원칙은 유지하되, 이미 작업이 진행된 경우 `#태스크정리`에서 Issue, 브랜치, 세션명, PR 연결, Backlog 분류를 함께 보정하는 보조장치가 필요하다.

## 3. 기대 효과

- 업무 태스크는 시작 절차로 정렬하는 원칙을 유지한다.
- 중간에 발생한 백로그와 번외사항을 작업 흐름 안에서 잃지 않는다.
- 사후 정리 시 누락된 Issue, 브랜치명, 세션명, PR 연결을 보정할 수 있다.
- 절차 누락이 있더라도 `#태스크정리` 단계에서 추적 가능한 상태로 회복할 수 있다.

## 4. 처리 기준

`STA-002`, `POL-003`에서 다음 기준을 검토한다.

- 업무 태스크는 기본적으로 `#태스크시작`으로 시작한다.
- `#태스크시작` 없이 진행된 작업이 있으면 `#태스크정리`에서 사후 보정한다.
- 사후 보정 항목은 Issue 존재 여부, 브랜치명, 세션명, 작업 범위, 제외 범위, PR 연결, Backlog 분류를 포함한다.
- 사후 보정이 가능하더라도 큰 작업을 Issue 없이 계속 진행하지 않는다.
- 중간 항목은 현재 Issue 반영, `[번외]`, Backlog, 새 Issue 후보 중 하나로 귀속한다.
