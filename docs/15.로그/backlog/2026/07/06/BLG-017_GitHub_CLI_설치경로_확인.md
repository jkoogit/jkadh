## 1. 내용

GitHub CLI(`gh`) 설치 경로를 확인하고, 개발 도구 관리 기준에 맞게 설치한다.

개발 프로그램은 가능한 `D:/dev/bin` 아래 적절한 폴더에서 관리하므로, 원격 작업 환경이 아니라 노트북 로컬 작업 환경에서 실제 경로를 확인한 뒤 설치를 진행한다.

## 2. 발생 배경

`#태스크정리` 중 PR 생성 흐름에서 `gh`가 설치되어 있지 않아 GitHub 앱 도구로 PR을 생성했다. 이후 `gh` 설치를 시도했으나, 사용자 프로필 경로가 아니라 `D:/dev/bin` 아래 관리하는 것이 운영 기준에 맞다는 사용자 지시가 있었다.

설치 시도 중 다음 경로에 파일이 생성되었을 수 있다. 다음 작업에서 실제 존재 여부와 정리 여부를 확인한다.

- 설치 시도 경로: `C:\Users\jkoog\AppData\Local\Programs\GitHub CLI`
- 다운로드 임시 파일: `C:\Users\jkoog\AppData\Local\Temp\gh_2.96.0_windows_amd64.zip`

## 3. 기대 효과

- 매번 `gh` 미설치로 PR 생성 절차가 지연되는 문제를 줄인다.
- 개발 도구 설치 위치를 일관되게 관리한다.
- Codex 작업에서 GitHub CLI를 사용할 수 있어 PR 확인, 인증 상태 확인, 브랜치/PR 조회 절차가 단순해진다.

## 4. 처리 기준

- 설치 전 `D:/dev/bin` 아래 기존 도구 관리 구조를 확인한다.
- GitHub CLI 설치 대상 폴더를 사용자에게 확인한다.
- 설치 후 `gh --version`과 `gh auth status`를 확인한다.
- 인증이 필요하면 사용자가 직접 인증할 수 있도록 절차를 분리한다.
- 설치 과정에서 사용자 프로필 또는 임시 폴더에 생성된 파일이 있다면 정리 여부를 사용자에게 확인한다.
