## 1. 내용

브랜치, 커밋, PR, 리뷰, 승급, 릴리즈 규칙을 별도 정책 문서로 정리한다.

## 2. 발생 배경

문서관리방안 작성 중 브랜치 전략이 함께 논의되었으나, 브랜치 상세 규칙은 문서관리방안보다 Git 작업관리 정책에 가깝다고 판단했다.

## 3. 기대 효과

- 한 Issue, 한 task 브랜치, 한 PR 흐름을 명확히 한다.
- PR 머지 후 새 task 브랜치로 이어가는 원칙을 고정한다.
- dev, stg, main 승급 절차를 문서화한다.

## 4. 처리 기준

Issue로 승격할 때 `POL-003_Git_작업관리방안.md` 작성을 범위로 한다.
