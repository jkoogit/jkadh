## 1. 내용

JKADH에서 CTO, 기획, 설계, 구현, 리뷰, QA, 리서치, 하네스, 운영 에이전트가 맡는 공통 역할과 역할별 운영규칙을 정리한다.

## 2. 발생 배경

GPT 세션과 Codex 세션에서 특정 AI 제품명보다 역할 기준으로 대화와 작업을 이어가야 한다는 방향이 논의되었다. 에이전트별 문서에서 반복되는 기본 책임은 공통역할로 분리할 필요가 있다.

## 3. 기대 효과

- 역할별 책임 경계를 명확히 한다.
- 같은 판단을 반복하지 않도록 역할 기준을 남긴다.
- 에이전트별 문서에서 반복되는 내용을 줄인다.
- CTO 대화운영규칙과 Git 작업관리방안의 전제가 된다.

## 4. 처리 기준

`ORG-001_에이전트_공통역할.md`와 `ORG-002_에이전트_역할별_운영규칙.md` 작성으로 해결한다.
