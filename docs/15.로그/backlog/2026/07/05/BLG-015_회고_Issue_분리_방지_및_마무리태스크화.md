## 1. 내용

세션 회고를 별도 Issue로 분리하지 않고, 작업 중인 Issue의 마무리 태스크로 처리하는 기준을 정리한다.

다음 세션부터는 회고 문서 작성 또는 회고 요약이 필요하더라도 별도 회고 Issue를 만들지 않는다. 현재 작업 Issue 안에서 `#세션마무리` 또는 `#태스크정리`의 마무리 태스크로 회고를 작성하고, 필요한 경우 같은 PR 또는 후속 정리 PR에 포함한다.

## 2. 발생 배경

Harness 자율성 단계 정리 세션을 마무리하면서 회고 문서 작성을 위해 별도 Issue를 생성했다. 그러나 회고는 작업 결과를 정리하는 종료 활동에 가깝기 때문에 매번 독립 Issue로 분리하면 작업 흐름이 늘어나고 세션 종료 절차가 과해질 수 있다.

회고가 큰 정책 변경이나 별도 분석 작업으로 커지는 경우가 아니라면, 작업 중인 Issue의 마지막 태스크로 정리하는 방식이 더 자연스럽다.

## 3. 기대 효과

- 세션 종료 절차가 짧고 일관된다.
- 회고가 원래 작업 Issue의 맥락 안에 남는다.
- 회고만을 위한 Issue와 PR이 불필요하게 늘어나는 것을 줄인다.
- 다음 세션 시작 시 작업 Issue, PR, 회고의 연결 관계를 더 쉽게 추적할 수 있다.

## 4. 처리 기준

`STA-002`, `POL-002`, `POL-003`에서 다음 기준을 정리한다.

- `#세션마무리` 시 회고는 기본적으로 현재 작업 Issue의 마무리 태스크로 진행한다.
- 회고를 위한 별도 Issue는 만들지 않는다.
- 회고 문서가 필요하면 현재 작업 Issue의 범위에 포함하거나 `[번외]` 마무리 태스크로 표시한다.
- 회고가 독립 분석, 운영 정책 변경, 장기 개선 작업으로 커지는 경우에만 Backlog 또는 새 Issue 후보로 분리한다.
- 회고 결과에는 다음 세션 권장 시작점과 남은 Backlog 판단을 포함한다.
