## 1. 내용

작업 성격과 난이도에 따라 어떤 에이전트 역할과 AI 개발자를 배치할지 판단하는 Capability Matrix를 정리한다.

## 2. 발생 배경

PRJ-002는 역할 기반 AI 배치를 목표로 정의했다. 실제 운영을 위해서는 역할, 강점, 금지 작업, 검증 필요 수준, 적합도 기준이 필요하다.

## 3. 기대 효과

- 작업 성격별 에이전트 배치 기준을 만들 수 있다.
- 특정 AI 제품명보다 역할과 Capability 중심으로 운영할 수 있다.
- 리뷰와 QA 보강이 필요한 작업을 사전에 구분할 수 있다.

## 4. 처리 기준

설계 문서에서 역할별 Capability, 적합 작업, 부적합 작업, 위험 조건, 검증 보강 기준을 표로 정의한다.
