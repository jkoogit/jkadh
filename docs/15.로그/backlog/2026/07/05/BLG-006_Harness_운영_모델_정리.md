## 1. 내용

Issue가 등록되었을 때 Harness가 작업을 분석하고, 에이전트 후보를 선정하고, 결과를 수집하고, 후속 흐름으로 연결하는 운영 모델을 정리한다.

## 2. 발생 배경

PRJ-002에서 Harness가 Issue 기반 작업 흐름을 운영한다는 목표를 정의했지만, 상태 전이와 책임 경계는 별도 설계 문서가 필요하다.

## 3. 기대 효과

- Harness와 AI 개발자의 책임 경계를 명확히 한다.
- Issue Loop, PR Review Loop, Report Loop의 연결 구조를 정의할 수 있다.
- 수동 Harness에서 부분 자동화로 넘어가는 기준을 만들 수 있다.

## 4. 처리 기준

`DSN-001` 또는 별도 Harness 설계 문서에서 입력, 출력, 상태 전이, 중단 조건, 승인 조건을 정의한다.
