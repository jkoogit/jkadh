## 1. 내용

회고 문서가 Codex 채팅세션과 직접 매핑되도록 회고 문서 Header, 명명규칙, 관련 템플릿을 보강한다.

현재 Codex 채팅창 세션명은 `009`처럼 채팅 도구 안의 순번을 포함할 수 있다. 그러나 다른 AI 에이전트도 자체 세션 번호를 사용할 수 있으므로, 회고 문서에는 세션 번호만이 아니라 `Agent` 또는 `작업 도구` 기준 식별자를 함께 남겨야 한다.

## 2. 발생 배경

RET-008 회고 문서는 `문서 ID`, `세션명`, `Issue`, `PR` 기준으로 추적할 수 있지만, Codex 채팅세션 번호와 직접 연결되는 필드는 없다.

채팅세션 번호를 회고 문서에 남기면 나중에 대화 원문을 찾기 쉬워진다. 다만 `009` 같은 번호만 저장하면 Codex, Claude, Gemini, Copilot 등 다른 에이전트의 세션 번호와 충돌할 수 있다.

따라서 회고 문서의 세션 종속성을 명확히 하려면 `Agent + 채팅세션 번호 + 세션명 + Issue/PR`을 함께 보존하는 방식이 필요하다.

## 3. 보완 방향

다음 항목을 회고 문서 Header 또는 템플릿에 추가하는 방안을 검토한다.

| 후보 필드 | 예시 | 목적 |
|---|---|---|
| 작업 도구 | `Codex` | 어느 AI 도구의 세션인지 식별 |
| Agent | `Codex/GPT-5/CTO` | 같은 도구 안에서도 역할과 모델 맥락 식별 |
| 채팅세션 번호 | `009` | 도구 내부 채팅세션 번호 기록 |
| 채팅세션 키 | `Codex-009` | Agent와 번호를 결합한 사람이 읽기 쉬운 키 |
| 세션명 | `058_Harness_태그기반_프로세스_실행경계_정리` | 저장소 작업 단위와 연결 |
| 기준 Issue | `#58` | GitHub 작업 단위와 연결 |
| 관련 PR | `#59`, `#60`, `#61`, `#62` | 실제 반영 이력 연결 |

명명규칙은 파일명에 채팅세션 번호를 바로 넣기보다 Header 필드로 관리하는 방안을 우선 검토한다. 파일명은 문서 ID와 회고 주제를 안정적으로 유지하고, 도구별 세션 번호는 Header와 인덱스에서 검색 가능하게 두는 편이 충돌과 이름 변경 부담이 작다.

## 4. 채번 기준 검토 후보

회고 문서와 세션 추적에 사용할 수 있는 채번 기준은 다음과 같다.

| 후보 | 예시 | 장점 | 우려 |
|---|---|---|---|
| RET 문서 ID 기준 | `RET-008` | 저장소 문서 기준으로 안정적이며 중복이 적다. | 채팅세션 번호와 직접 연결되지 않는다. |
| Agent + 채팅세션 번호 | `Codex-009` | 실제 대화창을 찾기 쉽고 다른 에이전트와 충돌을 줄인다. | 도구가 세션 번호를 안정적으로 제공하지 않으면 수동 입력에 의존한다. |
| Issue 번호 기준 | `ISSUE-058` | GitHub가 원자적으로 채번해 충돌이 적다. | Issue 없이 남기는 회고나 Backlog에는 직접 적용하기 어렵다. |
| 날짜 + Agent + 일련번호 | `20260712-Codex-001` | 여러 에이전트 동시 작업에 강하다. | 사람이 읽기에는 길고, 기존 RET 번호와 중복 관리가 생긴다. |
| 복합 키 | `RET-008 / Codex-009 / #58` | 문서, 대화, GitHub 작업을 모두 연결한다. | Header 표준과 인덱스 관리가 필요하다. |

초기 권장안은 `RET 문서 ID`를 기본 문서 ID로 유지하고, Header에 `Agent`, `채팅세션 번호`, `채팅세션 키`를 추가하는 복합 매핑 방식이다.

## 5. 처리 기준

- 회고 문서 Header 표준에 `Agent`, `채팅세션 번호`, `채팅세션 키`를 추가할지 결정한다.
- `docs/12.회고/README.md` 인덱스에 채팅세션 키 컬럼을 추가할지 검토한다.
- 회고 문서 템플릿이 있다면 해당 필드를 반영한다.
- 회고 문서 파일명에는 채팅세션 번호를 포함할지 여부를 별도 판단한다.
- 기존 RET 문서에 과거 세션 정보를 소급 반영할지 결정한다.
- BLG-019에서 다룬 채번충돌 방지 기준과 중복되지 않도록 문서 ID, Issue 번호, 채팅세션 번호의 역할을 분리한다.

## 6. 기대 효과

- 회고 문서에서 실제 Codex 채팅세션을 더 쉽게 찾을 수 있다.
- 다른 AI 에이전트의 세션 번호와 충돌하지 않도록 Agent 기준을 함께 보존할 수 있다.
- RET 문서 ID, GitHub Issue/PR, 채팅세션 번호가 각각 어떤 추적 책임을 갖는지 명확해진다.
- 다음 Harness 구현에서 세션 메타데이터 수집과 보고 형식을 더 일관되게 설계할 수 있다.
