## 1. 내용

스킬로 정의한 기능에는 실행 체크리스트를 만들고, 필수사항을 실행조건으로 명시하는 기준을 보강한다.

특히 `#태스크시작`, `#태스크정리`, `#태스크승급`처럼 저장소 상태와 문서 추적성을 바꾸는 스킬형 절차는 시작 전에 필수 조건을 확인하고, 조건이 비어 있으면 진행하지 않도록 한다.

## 2. 발생 배경

BLG-005 작업에서 `#태스크시작` 명령이 있었지만 GitHub Issue를 먼저 만들지 않고 PR을 생성한 뒤 사후 보정했다.

또한 `#태스크승급` 명령이 들어왔을 때 PR 렌더링 확인이라는 완료 조건이 남아 있었는데도 승급 절차로 진행하려 했다.

이는 스킬형 절차에 체크리스트와 실행조건이 명확히 고정되어 있지 않아 작업자가 상황에 따라 흔들릴 수 있음을 보여준다.

## 3. 기대 효과

- 태스크 시작 전에 Issue, Backlog, 브랜치, 범위, 제외 범위, 완료 조건, 검증 방법을 빠뜨리지 않는다.
- PR 생성 전에 연결 Issue와 Backlog 이력을 확인한다.
- 승급 전에 완료 조건과 리뷰/렌더링 확인 여부를 확인한다.
- 스킬형 절차가 보고 형식이 아니라 실행 게이트로 작동한다.

## 4. 처리 기준

다음 기준을 문서화할지 검토한다.

- 각 스킬형 절차에는 실행 전 체크리스트를 둔다.
- 체크리스트 항목 중 필수사항은 실행조건으로 분리한다.
- 필수 실행조건이 비어 있으면 작업을 중단하고 사용자 확인 또는 사후 보정을 먼저 수행한다.
- 태스크 시작 시 Issue가 없으면 생성하거나 생략 사유를 명시한다.
- PR 생성 전에는 PR 본문, Backlog 파일, 후속 Backlog의 연결 Issue/PR을 확인한다.
- 승급 전에는 PR 목적, 완료 조건, 리뷰/렌더링 확인, 검증 결과, 연결성 게이트를 확인한다.
