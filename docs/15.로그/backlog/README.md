## 인덱스 기준

- 미해결 Backlog만 이 인덱스에 둔다.
- 개별 Backlog 파일은 생성일 기준 날짜 폴더에 고정한다.
- 해결 상태, 연결 Issue, 연결 PR, 해결 문서는 개별 Backlog 파일의 연결 이력에 남긴다.

## 상태

| 상태 | 한글명 | 의미 | 처리 기준 |
|---|---|---|---|
| Open | 등록 | 등록만 되었고 아직 분류하지 않음 | Issue 종료 시 정리한다. |
| Ready | 준비 | 바로 Issue로 승격 가능 | 다음 Issue 선정 시 후보로 검토한다. |
| Blocked | 차단 | 선행 Issue, 결정, 문서가 필요함 | 의존 대상 해결 후 재검토한다. |
| Deferred | 보류 | 필요하지만 당장 우선순위가 낮음 | 정기 점검 시 재검토한다. |
| Issue Linked | 이슈 연결 | GitHub Issue로 승격됨 | 연결 Issue와 PR 상태를 추적한다. |
| Resolved | 해결 | 해결됨 | 미해결 인덱스에서 제거한다. |
| Rejected | 채택 제외 | 채택하지 않음 | 미해결 인덱스에서 제거한다. |

## 처리시점

| 처리시점 | 의미 |
|---|---|
| 즉시 | 현재 Issue 범위를 해치지 않고 바로 처리 가능 |
| Issue 종료 시 | 현재 Issue 완료 전 정리 필요 |
| 다음 Issue 선정 시 | 다음 작업 후보로 검토 |
| 선행 작업 완료 후 | 의존 대상이 해결되면 재검토 |
| 정기 점검 시 | 장기 보류 항목 점검 |
