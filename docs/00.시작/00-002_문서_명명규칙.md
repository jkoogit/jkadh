# 00-002 문서 명명규칙

> 문서번호: 00-002  
> Version: v0.2 (Draft)  
> 상태: Draft  
> Related Work Order: [WO-002 문서 명명규칙 및 링크 규칙 수립](https://github.com/jkoogit/jkadh/issues/3)

## 1. 목적

본 문서는 JKADH 저장소에서 사용하는 문서번호, 문서명, 폴더명, 문서 제목, 문서 간 링크 규칙을 정의한다.

명명규칙은 사람이 보기 위한 정리 방식이 아니라, AI가 저장소를 일관되게 읽고 작업하기 위한 Project Loading Order의 일부로 사용한다.

## 2. 기본 원칙

- 모든 공식 문서는 문서번호를 가진다.
- 문서명은 한글을 기본으로 한다.
- 공백은 사용하지 않고 `_`를 사용한다.
- 파일 확장자는 `.md`를 사용한다.
- 문서명은 역할이 바로 드러나게 작성한다.
- 약어만으로 된 파일명은 피한다.
- 같은 개념은 항상 같은 이름으로 부른다.
- 문서 끝에는 작업 이력을 기록한다.

## 3. 폴더명 규칙

문서 폴더는 번호와 한글명을 함께 사용한다.

예:

```text
00.시작
01.프로젝트
02.정책
03.조직
04.현재상태
05.로드맵
06.기획
07.설계
08.ADR
09.구현
10.운영
11.학습
12.템플릿
13.용어
14.실험
```

번호는 단순 정렬용이 아니라 AI가 프로젝트를 읽는 공식 순서인 Project Loading Order를 의미한다.

## 4. 문서번호 규칙

문서번호는 다음 형식을 사용한다.

```text
폴더번호-문서순번
```

예:

```text
00-001
00-002
01-001
02-001
07-001
```

문서번호는 문서의 논리적 위치와 읽기 순서를 나타낸다.

## 5. 파일명 규칙

파일명은 다음 형식을 기본으로 한다.

```text
문서번호_문서명.md
```

예:

```text
00-001_저장소_청사진.md
00-002_문서_명명규칙.md
01-001_프로젝트헌장.md
02-001_개발표준.md
02-002_거버넌스.md
13-001_용어사전.md
```

## 6. README 규칙

각 주제 폴더에는 문서 인덱스 역할을 하는 `README.md`를 둔다.

README는 다음 내용을 포함한다.

- 폴더 목적
- 문서 목록
- 문서번호
- 문서 상태
- 관련 문서 링크

README 자체는 폴더 인덱스이므로 `00-000`, `01-000`처럼 문서번호를 부여할 수 있다.

## 7. 문서 제목 규칙

문서의 첫 번째 제목은 문서번호와 문서명을 포함한다.

예:

```markdown
# 00-001 저장소 청사진 (Repository Blueprint)
# 02-001 개발 표준
```

영문명은 필요한 경우 제목 또는 본문에서 병기한다.

## 8. 문서 간 링크 규칙

문서 간 참조는 GitHub 상대경로를 사용한다.

예:

```markdown
[00-001 저장소 청사진](./00-001_저장소_청사진.md)
[01-001 프로젝트 헌장](../01.프로젝트/01-001_프로젝트헌장.md)
[02-002 거버넌스](../02.정책/02-002_거버넌스.md)
```

문서가 아직 존재하지 않는 경우에도 확정된 경로가 있다면 링크를 먼저 작성할 수 있다.

## 10. 작업 이력 규칙

모든 공식 문서 끝에는 다음 형식의 작업 이력을 둔다.

```markdown
---

## 작업 이력

| 작업일시 | 작업 에이전트 | 작성자 | 내용 한 줄 요약 |
|---|---|---|---|
| 2026-07-03 | TIO(ChatGPT) | jk / TIO | 문서 작성 |
```

## 11. 변경 원칙

명명규칙 변경은 전체 문서 링크에 영향을 줄 수 있으므로 신중하게 수행한다.

명명규칙 변경이 필요한 경우 다음 절차를 따른다.

1. 변경 사유 기록
2. 영향 문서 확인
3. 링크 경로 수정
4. 필요 시 ADR 작성
5. Product Owner 또는 TIO 검토
