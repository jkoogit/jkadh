# JKADH 저장소 청사진 (Repository Blueprint)

> Version: v0.1 (Draft)

## 목적

본 문서는 JKADH 저장소의 최상위 구조와 각 영역의 역할을 정의한다.

저장소 구조는 AI와 사람이 동일한 기준으로 프로젝트를 이해하고 운영하기 위한 공식 구조이다.

## Repository 구성

- docs/
- skills/
- tools/
- runtime/
- storage/
- examples/
- .github/

## 성장 원칙

Docs → Skills → Tools → Runtime

문서로 검증되지 않은 기능은 Runtime으로 구현하지 않는다.

## Epic-001 범위

- Repository 구조
- 핵심 정책 문서
- 조직 정의
- 문서 체계
- Meta Model

Runtime 구현은 Epic-001 범위에 포함하지 않는다.
