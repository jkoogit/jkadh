# 문서 명명규칙

> Version: v0.1 (Draft)  
> Related Work Order: [WO-002 문서 명명규칙 및 링크 규칙 수립](https://github.com/jkoogit/jkadh/issues/3)

## 1. 목적

본 문서는 JKADH 저장소에서 사용하는 문서명, 폴더명, 문서 제목, 문서 간 링크 규칙을 정의한다.

명명규칙은 사람이 보기 위한 정리 방식이 아니라, AI가 저장소를 일관되게 읽고 작업하기 위한 Project Loading Order의 일부로 사용한다.

## 2. 기본 원칙

- 문서명은 한글을 기본으로 한다.
- 공백은 사용하지 않고 `_`를 사용한다.
- 파일 확장자는 `.md`를 사용한다.
- 문서명은 역할이 바로 드러나게 작성한다.
- 약어만으로 된 파일명은 피한다.
- 같은 개념은 항상 같은 이름으로 부른다.

## 3. 폴더명 규칙

문서 폴더는 번호와 한글명을 함께 사용한다.

예:

```text
00.시작
01.프로젝트
02.정책
03.조직
04.현재상태
05.로드맵
06.기획
07.설계
08.ADR
09.구현
10.운영
11.학습
12.템플릿
13.용어
14.실험
```

번호는 단순 정렬용이 아니라 AI가 프로젝트를 읽는 공식 순서인 Project Loading Order를 의미한다.

## 4. 파일명 규칙

파일명은 다음 형식을 기본으로 한다.

```text
개념명.md
개념명_세부항목.md
```

예:

```text
프로젝트헌장.md
개발표준.md
거버넌스.md
용어사전.md
저장소_청사진.md
문서_명명규칙.md
```

## 5. 문서 제목 규칙

문서의 첫 번째 제목은 파일명과 동일하거나 동일한 의미를 가진다.

예:

```markdown
# 프로젝트 헌장
# 개발 표준
# 저장소 청사진
```

영문명은 필요한 경우 제목 또는 본문에서 병기한다.

예:

```markdown
# 저장소 청사진 (Repository Blueprint)
```

## 6. 문서 간 링크 규칙

문서 간 참조는 GitHub 상대경로를 사용한다.

예:

```markdown
[저장소 청사진](./저장소_청사진.md)
[프로젝트 헌장](../01.프로젝트/프로젝트헌장.md)
[거버넌스](../02.정책/거버넌스.md)
```

문서가 아직 존재하지 않는 경우에도 확정된 경로가 있다면 링크를 먼저 작성할 수 있다.

## 8. 변경 원칙

명명규칙 변경은 전체 문서 링크에 영향을 줄 수 있으므로 신중하게 수행한다.

명명규칙 변경이 필요한 경우 다음 절차를 따른다.

1. 변경 사유 기록
2. 영향 문서 확인
3. 링크 경로 수정
4. 필요 시 ADR 작성
5. Product Owner 또는 TIO 검토
