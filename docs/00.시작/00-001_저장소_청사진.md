# 00-001 저장소 청사진 (Repository Blueprint)

> 문서번호: 00-001  
> Version: v0.2 (Draft)  
> 상태: Draft  
> 상위 작업: [TASK-001 Foundation 핵심 문서 구축](https://github.com/jkoogit/jkadh/issues/2)

## 1. 목적

본 문서는 JKADH 저장소의 최상위 구조와 각 영역의 역할을 정의한다.

저장소 구조는 AI와 사람이 동일한 기준으로 프로젝트를 이해하고 운영하기 위한 공식 구조이다.

모든 문서와 구현은 본 청사진을 기준으로 작성한다.

## 2. 저장소 목표

JKADH는 단순한 문서 저장소가 아니다.

다음 요소를 포함하는 AI Development Runtime Platform을 구축한다.

- Docs
- Skills
- Tools
- Runtime
- Storage
- Examples

## 3. Repository 구성

```text
docs/
skills/
tools/
runtime/
storage/
examples/
.github/
```

### docs

조직의 정책과 설계를 관리한다.

대표 문서:

- Project Charter
- Development Standard
- Governance
- Glossary
- ADR
- Architecture

### skills

AI가 반복 수행하는 표준 작업 절차를 관리한다.

예:

- Session Start
- Session End
- Branch Promotion
- Resume Create
- Promotion Review

### tools

사람과 AI가 사용하는 CLI 및 운영 도구를 관리한다.

예:

- Resume Manager
- Work Order Generator
- Cost Monitor
- GitHub Sync

### runtime

JKADH Runtime 구현 영역이다.

예:

- Router
- Scheduler
- Event Listener
- Rule Engine
- Resume Manager
- Passport Manager

### storage

Runtime이 사용하는 상태 저장소의 구조를 정의한다.

예:

- Work Order
- Resume
- Passport
- Metrics
- Audit

### examples

JKADH를 실제 적용하는 프로젝트를 관리한다.

첫 번째 Reference Project:

- PDFowers

## 4. Repository 성장 원칙

Repository는 다음 순서로 성장한다.

```text
Docs
↓
Skills
↓
Tools
↓
Runtime
```

문서로 검증되지 않은 기능은 Runtime으로 구현하지 않는다.

## 5. Epic-001 범위

Epic-001에서는 다음 항목만 구축한다.

- Repository 구조
- 핵심 정책 문서
- 조직 정의
- 문서 체계
- Meta Model

Runtime 구현은 Epic-001의 범위에 포함하지 않는다.
